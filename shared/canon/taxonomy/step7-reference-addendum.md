### Post-mortem: Step 5b filter gap

Step 5b iterated on `'Lists' = ANY(a.path_parts) AND (name ILIKE '%lists%' OR name = 'Species and specimens')`. That path-filter excluded the two Reference parked L2 shells (`reference-general-reference-lists` and `reference-research-tools-and-resources`) because their paths don't contain a `Lists` element — they're rooted directly in `Reference / *`. Result: the empty `research-tools-and-resources` shell stuck around after Step 5b until this follow-up TX cleaned it up.

**Lesson for future iterative-drain passes:** the eligibility filter must enumerate every known shell-host, not just the Lists-tree. For this codebase that means:
```sql
WHERE (
  'Lists' = ANY(a.path_parts)                                        -- Lists-tree shells
  OR a.id IN ('reference-general-reference-lists',                   -- Reference parked L2 shells
              'reference-research-tools-and-resources')
  OR /* any future parked-L2 shells from new disposition passes */
)
```
Or — cleaner — define an explicit shell-eligibility set up-front rather than inferring from path/name patterns. When a future disposition pass introduces a new parked-L2 set, add it to the eligibility list at the same time, so iterative drain auto-handles it.
